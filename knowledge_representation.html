TEI/XML document

Excerpts of the Wikipedia article on Chiune Sugihara have been encoded as a TEI/XML document. Selected entities are annotated inline using elements such as <persName>, <placeName>, <orgName>, and <rs>. Each entity is described in the back matter using dedicated list elements (listPerson, listPlace, listOrg, listBibl, listEvent, listObject) and reconciled to authority files including Wikidata, VIAF, and OCLC. Quotation is also marked up using <quote>.

TEI → HTML (XSLT)

The TEI/XML document is transformed into a HTML page using a Python script built on the xml.etree.ElementTree library. The script reads the back matter to build an index of entities, then processes the body text, wrapping each annotated entity in a colour-coded span linked to its Wikidata entry. The output also includes an entity index table and a relations table derived from listRelation.

Transformed HTML file

The output of the TEI to HTML transformation is presented below. Entities are highlighted in colour according to their type: red for persons, blue for places, green for bibliographic works, light green for organisations, pink for objects (event, concept, infrastructure), and purple for a textual phenomenon. Each highlighted entity links directly to its corresponding Wikidata entry.

TEI → RDF

The TEI/XML document is transformed into an RDF dataset in Turtle format using a Python script built on the rdflib library. The script maps each entity to the class defined in the conceptual model (e.g. foaf:Person, schema:City, crm:E5_Event, skos:Concept) and each relation to the corresponding property (e.g. schema:workLocation, crm:P15_was_influenced_by). Every entity is linked to its Wikidata counterpart via owl:sameAs where an exact match exists. Where no precise Wikidata entry is available — as in the case of Transit Visa — a broader concept is indicated using crm:P2_has_type instead.
